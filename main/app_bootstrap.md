# `app_bootstrap` 说明

这个模块负责把原本堆在 `app_main()` 里的启动装配逻辑拆出来。
它的目标不是增加新能力，而是让启动顺序更可读、更容易验证。

## 核心职责

1. 收敛系统启动阶段的初始化步骤。
2. 让 `app_main()` 更像总入口，而不是所有细节都写在一处。
3. 给启动链路留出更稳定的验证点和测试点。

## 为什么它重要

启动链路通常最怕两件事：

- 顺序改乱了，但肉眼不容易发现
- main 函数越堆越长，最后没人敢动

`app_bootstrap` 的价值就在于把这些步骤明确分层，让“启动本身”也成为一个可维护模块。
