# `boards` 模块说明

这个目录负责板型分发和板级差异收敛。
上层只需要调用统一的 `board_*` 接口，不用关心当前编译目标到底是哪块板子。

## 当前结构

- `board_dispatch.c`
  统一分发入口，把通用 `board_*` 调用转到具体板型实现。
- `board_ws_175_amoled.c`
  WS175 AMOLED 板型实现，包含 QSPI 面板、触摸、共享 I2C 总线等逻辑。
- `board_ws_185.c`
  WS185 板型实现，走另一套 LCD、触摸和背光驱动。
- `board_api.h`
  上层统一使用的板级接口定义。

## 统一接口覆盖的职责

1. 板级初始化。
2. 显示上下文初始化。
3. 注册刷屏完成回调。
4. 设置亮度。
5. 提供板卡静态信息。
6. 导出共享 I2C 总线。

## 维护建议

- 新增板型时，优先保持 `board_api.h` 不变，只在这个目录扩展具体实现。
- 如果显示旋转、触摸映射、亮度命令不同，也应尽量留在板型实现内收敛。
- 上层业务和 UI 最好只依赖统一 `board_*` 接口，不要直接引用某块板子的私有实现。
