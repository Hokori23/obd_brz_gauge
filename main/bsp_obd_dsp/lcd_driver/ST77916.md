# `ST77916.c/.h` 阅读说明

这是项目侧的 LCD 封装，不是最底层的 panel 驱动。

## 它处在什么层级

```text
app_main / LVGL
   -> ST77916.c
      -> esp_lcd_st77916.c
         -> ESP-IDF esp_lcd / SPI / QSPI
```

所以它更像一个“板级显示装配器”。

## 它做了什么

1. 保存一大段屏幕初始化命令表。
2. 负责 LCD reset。
3. 初始化底层显示总线和 panel。
4. 初始化背光。
5. 初始化触摸。
6. 对外暴露 `LCD_Init()` 和 flush 回调设置接口。

## 为什么 `LCD_Init()` 同时带了触摸

因为对这个项目来说，“屏幕子系统”不只是显示面板，还包括：

- panel
- 背光
- 触摸

这种合并在板级工程里很常见。

## 你读这份文件时重点看

- `LCD_SetFlushCallback()`
- `ST77916_Reset()`
- `LCD_Init()`
- `QSPI_Init()`

## 那一大串初始化命令是什么

那是面板厂商要求的寄存器配置表。可以把它理解成“屏幕芯片启动脚本”。

通常你不需要逐条理解，只要知道：

- 这是硬件 bring-up 必需部分
- 一旦屏幕能正常显示，除非换屏，不会经常改
