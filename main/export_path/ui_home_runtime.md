# `ui_home_runtime` 模块说明

这是首页运行时控制模块。
核心职责：首页分页切换、懒加载刷新、编辑态切换、页面可见性调度。

## 模块负责什么

- 管理首页分页容器的创建与切换
- 只刷新当前页或相邻页，避免整屏无差别重绘
- 协调编辑态、浏览态与运行时数据刷新
- 串联 `ui_home_runtime_widgets`，把布局结果挂到具体页面

## 为什么单独拆出来

首页是整个项目里最频繁刷新的 UI 区域之一。
如果把分页、状态切换、数据刷新都堆在 `ui.c` 或单个 screen 文件里，后续很难判断某次刷新到底是布局逻辑、业务数据，还是交互状态导致的。

## 阅读建议

1. 先看 `ui_home_pager.*` 了解分页容器
2. 再看 `ui_home_runtime.c` 里的状态切换和刷新入口
3. 最后看 `ui_home_runtime_widgets.*` 理解具体卡片是怎么挂载的
