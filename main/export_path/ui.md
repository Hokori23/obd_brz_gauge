# `ui.c/.h` 阅读说明

这组文件是整个 UI 层的总装配入口。

## `ui.c` 在做什么

1. 声明所有页面对象和大量全局控件指针。
2. 定义一些 UI 层共享状态。
3. 提供 `ui_init()`，把各页面初始化函数串起来。
4. 提供一些全局更新逻辑、页面切换逻辑和页面间共享行为。

## 为什么会有这么多全局变量

这是 LVGL + 设计工具导出代码的典型形态。

每个控件都需要一个可访问指针，便于：

- 页面刷新时改文本
- 回调时改样式
- 页面切换时直接引用

这和前端框架里的组件局部 state 很不一样。

## 你最应该关注的不是“控件创建”，而是这些内容

- 页面初始化顺序
- 默认页选择逻辑
- 是否从 `NVS` 读取设置
- 是否直接访问 `obd_data_cache`
- 哪些页面有周期刷新或告警逻辑

## 阅读建议

第一次读 `ui.c`，把它当成“页面注册中心”和“全局 UI 状态文件”就够了，不要试图逐个控件理解。
