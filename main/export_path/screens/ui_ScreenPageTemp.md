# `ui_ScreenPageTemp.c` 阅读说明

这是一个很典型的“纯数据显示页”。

## 页面目标

三行显示三类温度：

- `CLT` 水温
- `IAT` 进气温
- `OIL` 机油温

## 这份代码的特点

- UI 结构很清晰
- 主要是在创建控件和排版
- 业务逻辑很少

## 你读它时抓住两点就够了

### 1. 页面结构如何组织

通过：

- `make_row()`
- `make_hdiv()`

把重复 UI 结构抽出来。

这说明虽然是导出代码风格，作者还是做了一点手工整理。

### 2. 为什么保留旧变量名

文件最后把：

- `ui_LabelCoolantTempText`
- `ui_LabelIntakeTempText`
- `ui_LabelOilTempText`

重新指向新数组里的 label。

这是一种兼容旧更新逻辑的做法，避免改动太多其他代码。

## 你可以把它理解成

这是 LVGL 版的“展示组件”，真正的数据更新逻辑不在这里，而在别处定时把值写进这些 label。
