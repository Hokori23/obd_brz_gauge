# BRZ ZD8 OBD 协议快速诊断

由于 ZD8 是新一代 BRZ，其 OBD 报文可能不同于 ZC6（已知工作）。

## 症状
- ✗ BLE 已连接
- ✗ 所有参数显示 0（未收到任何数据）
- ✗ 车型为 BRZ ZD8

## 快速修复步骤

### 第一步：尝试所有协议
进入设置界面，按以下顺序依次尝试各协议（每次切换后重新连接OBD设备）：

1. **协议 1** - ISO 9141-2 (5 baud init)
2. **协议 2** - ISO 9141-2 (fast init)
3. **协议 3** - ISO 14230-4 KWP2000 (5 baud)
4. **协议 4** - ISO 14230-4 KWP2000 (fast init)
5. **协议 5** - ISO 15765-4 (CAN@250K) [最常见的现代协议]
6. **协议 6** - ISO 15765-4 (CAN@500K) [BRZ ZC6 已验证工作]
7. **协议 7** - ISO 15765-4 (CAN@arbitrary)
8. **协议 8-11** - 其他变体

### 第二步：判断成功标准

每次更改协议后：
1. 在 BLE SCAN 页面**重新连接** OBD 设备
2. 等待 3-5 秒，观察仪表主屏是否有任何数据变化
3. 特别观察：
   - **RPM** 应该显示当前转速（如：800-1200）
   - **Speed** 应该显示当前车速（如：0 or >0）

**成功信号**：
- RPM 不为 0
- Speed 数值有变化
- 其他参数（水温、油温等）有合理的数值

### 第三步：找到正确协议后

一旦找到有效协议（例如发现是协议 5），需要向项目报告：

```
嗨，我的 BRZ ZD8 用协议 [NUMBER] 成功读取数据。
这与 ZC6（用协议 6）不同。
```

然后我们可以：
1. 添加 BRZ ZD8 配置预设
2. 让后续用户自动选择

## 深层诊断（如果协议都不工作）

### 验证 OBD 设备本身

用手机测试同一 OBD 设备：
1. 下载 **Torque Pro**（付费 ~$5）或 **HaynesOBD**（免费）
2. 用手机蓝牙连接同一 OBD 设备
3. 扫描 PID `01 0C`（RPM）
   - ✓ 手机能读到 → OBD 设备正常，问题在仪表代码
   - ✗ 手机也读不到 → OBD 适配器故障或与车不兼容

### 检查仪表日志

连接后查看串口日志（USB 连接 + `idf.py monitor`），关键词：
- `elm327_ble: Send 01 0C` - 发送读 RPM 命令
- `elm327_ble: RAW[...]` - 收到响应
- `Parsed RPM` 或错误信息

**预期日志**（協議正确时）：
```
elm327_ble: Send 01 0C
elm327_ble: RAW[...]: 41 0C 12 34
elm327_ble: Parsed RPM: 4660
```

**异常日志**（協議错误时）：
```
elm327_ble: Send 01 0C
elm327_ble: RAW[...]: ?
elm327_ble: PARSE_FAIL or TIMEOUT_AGAIN
```

## 附件：通用 OBD-II PID 查询

标准兼容的 PID（几乎所有汽车都支持）：
| 命令 | 含义 | 单位 |
|-----|------|------|
| `01 0C` | RPM | rpm (divide by 4) |
| `01 0D` | Speed | km/h |
| `01 05` | Coolant Temp | °C (A-40) |
| `01 0F` | IAT | °C (A-40) |
| `01 04` | Engine Load | % |
| `01 11` | TPS | % |
| `01 42` | Battery Voltage | V |

如果至少有一个 PID 工作（如 RPM），说明协议是对的，只是 ZD8 特定 PID 不同。
