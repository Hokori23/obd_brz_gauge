# OBD 数据接收故障排查指南

当连接到 OBD 设备后无法读取数据，请按以下步骤诊断：

## 第一步：确认 BLE 物理连接

### 1.1 检查设备是否被识别

1. 在仪表中进入 **BLE SCAN** 页面
2. 检查您的 OBD 设备是否出现在扫描列表中
3. 记下设备显示的完整名称

**如果设备不出现：**
   - 确认 OBD 设备已打开并处于配对模式
   - 检查 OBD 设备的蓝牙是否已启用
   - 尝试重启 OBD 设备
   - 如果使用的是不同品牌/型号，可能需要调整扫描参数

### 1.2 检查连接状态

连接后检查以下方式确认已连接：
- 在仪表的 **版本页面** 顶部应显示连接状态指示
- 查看串口日志（如已接入USB）输出 `elm327_ble: OBD BLE connected`

**如果连接失败：**
   - 确认配对密码（如需要）
   - 检查 OBD 设备是否支持 BLE（而非仅支持 Bluetooth Classic）
   - 尝试在手机上连接该设备验证其正常性

---

## 第二步：验证协议连接

### 2.1 检查 ELM327 初始化

连接后，仪表应自动发送一系列初始化命令：
```
ATZ       # 复位
ATE0      # 关闭回显
ATL0      # 关闭换行符
ATS1      # 启用空格
ATH0      # 关闭报头
ATAT1     # 自适应时序
ATST 19   # 设置超时
ATSP?     # 设置协议（? 为您在设置中配置的值）
ATSH7E0   # 设置请求帧 ID
01 00     # 能力探测
```

如启用了串口输出，应在日志中看到这些命令。

**初始化失败的迹象：**
   - 日志中持续显示 "Send" 但无回应
   - 设备反复重试初始化
   - 无法进入数据轮询阶段

**解决方案：**
   1. 检查选定的协议是否与您的车适配
      - **Subaru BRZ（推荐）**：协议 6 (ISO 15765-4 / CAN@500K)
      - 其他车型可能需要协议 1-11（见 NVS 配置）
   2. 尝试更换协议后重新连接
   3. 使用手机 OBD 软件（如：Carista、Torque）验证 OBD 设备本身是否正常

---

## 第三步：检查数据轮询和解析

### 3.1 验证数据轮询流程

仪表连接后应循环查询以下 PID：
- `01 0C` - RPM
- `01 0F` - IAT (进气温度)
- `01 0D` - Speed (速度)
- `01 05` - Coolant Temp (水温)
- `01 04` - Engine Load (负荷)
- `01 11` - TPS (油门)
- `01 42` - Battery Voltage (电压)
- `21 01` 或 `22 10 17` - Oil Temp (油温，仅 BRZ)

### 3.2 从 UI 检查接收到的数据

1. 打开仪表主页面（显示 RPM、速度等）
2. 观察数值是否在变化
3. 记录看到的具体值（如所有参数都是 0，或显示特定错误值）

**常见的错误数据表示：**
| 参数 | 错误标记 | 含义 |
|-----|--------|------|
| RPM | 0 | 从未接收到有效数据 |
| Speed | 0 | 从未接收到或当前车速为 0 |
| Oil Temp | -100 | 未响应或 OBD 设备不支持 |
| Load | -1 | 从未接收到有效数据 |
| TPS | -1 | 从未接收到有效数据 |

### 3.3 逐个测试 PID 查询

如果仪表不显示数据，可使用手机级别的 OBD 应用（如 Torque、Carista）验证：
- 这些 PID 在您的车上是否被支持
- OBD 设备的回应格式是否标准

## 第四步：RS485 刹车温度（独立）

RS485 使用不同的硬件接口，故障独立排查：

### 4.1 验证硬件连接

检查 **已配置的 GPIO 引脚**（在 `menuconfig` 中）：
```
RS485_TX_GPIO: 13
RS485_RX_GPIO: 12
RS485_DE_RE_GPIO: -1 (禁用，因为 USB 模块自动处理)
```

确保：
- USB 转 RS485 模块已连接到 GPIO13（TX）和 GPIO12（RX）
- 模块与刹车温度传感器的数据线已接妥
- 传感器已上电

### 4.2 检查刹车温度状态

在 UI 的 **刹车温度页面** 顶部有状态指示：
- `OK` - 正常，已接收到温度数据
- `PROBE` - 正在探测中
- `TIMEOUT` - 传感器无响应（30 秒未收数据）
- `PARSE FAIL` - 响应格式错误

### 4.3 故障排除

**状态为 TIMEOUT：**
1. 确认 GPIO13/12 的接线正确
2. 检查 USB 转 RS485 模块的波特率设置（固定 9600 baud）
3. 验证传感器是否通电
4. 用 USB 转 TTL 模块直接从 PC 测试通信

**状态为 PARSE FAIL：**
- 传感器的 Modbus RTU 回应格式可能不标准
- 验证传感器的设置和固件版本

---

## 第五步：综合检查

### 5.1 重启整个系统

1. 断开仪表电源
2. 重启 OBD 设备
3. 重新打开仪表
4. 在 BLE SCAN 页面重新连接

### 5.2 检查 NVS 配置

进入设置页面，确认：
- **OBD 协议** - 与您的车型匹配
- **BLE 设备名** - 已保存正确的设备名称
- **亮度、主题** - 其他配置应正常

### 5.3 清除配置并恢复出厂设置

如长时间无法解决，可尝试：
1. 进入开发者菜单
2. 选择 **清除 NVS** 或 **恢复出厂设置**
3. 重新配置仪表

如需进一步调试，可启用详细日志：

```bash
idf.py menuconfig
# 导航到 Component config → Log output
# 将日志级别设为 DEBUG
idf.py build
```

### 7.2 通过串口监管查看日志


关键日志关键词：
- `elm327_ble:` - BLE 连接和数据相关
- `rs485_brake_temp` - 刹车温度传感器
- `obd_data_cache` - 数据缓存更新
- `ERROR` 或 `WARN` - 任何错误消息

## 常见问题解答

### Q：为什么屏幕上所有参数都显示 0？
**A：** 通常表示数据轮询还未开始，或 OBD 设备未正确初始化。检查 BLE 连接状态和日志中的初始化命令。

### Q：OBD 设备能连接但收不到新数据？
**A：** 
1. 确认协议选择正确（Subaru 应为协议 6）
2. 检查手机 OBD 应用是否能读取相同 PID
3. 尝试重新连接或更换 OBD 设备

### Q：速度和转速可以读取，但油温/负荷无法读取？
**A：** 这些 PID 在您的车型上可能不被支持，或需要不同的查询方式。参考 [车型文档](./README.zh-CN.md)。

### Q：如何判断 OBD 设备故障？
**A：** 在手机上安装标准 OBD 应用（如 Torque Pro，需付费），尝试连接同一 OBD 设备。若手机也收不到数据，则为设备故障。
