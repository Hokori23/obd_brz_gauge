# ZC6 G-force 路测与验收计划

更新时间：`2026-07-02`

## 1. 目的

这份文档用于补齐当前 `G-force-OBD` 与 `G-force-esp32` 的最后一段证据链。

当前仓库已经具备：

1. 按需请求指标的代码实现
2. `G-force-OBD` 的 `0x0D0` 监听链路
3. `G-force-esp32` 的 `QMI8658` 采样链路
4. G-force 圆形十字坐标 UI
5. 仓库内静态契约测试与整机编译通过
6. 节流后的实机诊断日志：
   - `GFORCE OBD sample ...`
   - `GFORCE IMU raw=... bias=... filt=...`

但还没有被仓库内自动化直接证明的部分是：

1. 目标车 `2019 ZC6` 上 `0x0D0` 是否持续稳定输出
2. 目标 ELM327 蓝牙适配器是否能稳定支持 `ATCRA0D0 + ATMA`
3. OBD 与 IMU 两条链路的横纵向正负方向是否都符合当前安装方向
4. 圆屏上的红点阻尼、黄区衰减和四象限语义是否最终满足体感

## 1.1 串口日志辅助

如果实车验收时串口可用，优先观察以下日志：

1. `elm327_ble`：
   - `Entered G-force OBD monitor mode for CAN 0x0D0`
   - `GFORCE OBD sample lat=... lon=... line=...`
2. `qmi8658_g`：
   - `QMI8658 ready`
   - `QMI8658 bias x=... y=...`
   - `GFORCE IMU raw=(...) bias=(...) filt=(...)`

它们的意义分别是：

1. `Entered G-force OBD monitor mode`：
   - 说明页面切换已经真的进入 `ATCRA0D0 + ATMA` 监听路径
2. `GFORCE OBD sample ...`：
   - 说明 `0x0D0` 监听行已经被成功解析成横纵向加速度
3. `QMI8658 bias ...`：
   - 说明 IMU 页面重新进入后已经做过零点采样
4. `GFORCE IMU raw/bias/filt ...`：
   - 可直接判断是原始值异常、bias 偏了，还是滤波后方向不对

## 2. 测试前提

路测前应满足：

1. 固件来自当前已编译通过版本
2. 已能正常进入首页并切到：
   - `G-force-OBD`
   - `G-force-esp32`
3. 车辆为 `ZC6 / FT86 gen1`
4. 板子固定方向在测试期间保持不变
5. 蓝牙已连接目标 ELM327
6. 车辆处于安全、合法、低风险测试环境

## 3. 先验预期

### 3.1 UI 先验

无论数据源来自 OBD 还是 IMU，页面都应满足：

1. 主视觉是圆形十字坐标图
2. 当前值由单个红点表达
3. 黄色区域表达历史峰值包络
4. 左上 / 右上 / 左下 / 右下四象限分别表示：
   - `ACC + LEFT`
   - `ACC + RIGHT`
   - `BRAKE + LEFT`
   - `BRAKE + RIGHT`

### 3.2 数据先验

静止状态下预期：

1. 红点应靠近圆心
2. 黄区应缓慢回落，不应永久挂在外圈
3. `G-force-OBD` 若车辆广播正常，应在进入页面后很快出现有效变化
4. `G-force-esp32` 若 IMU 正常，应在进入页面后完成 bias 复位并回到接近圆心

## 4. `G-force-OBD` 验收步骤

### 4.1 页面进入与模式切换

步骤：

1. 冷启动设备
2. 完成 BLE 连接
3. 切到 `G-force-OBD`
4. 停留至少 `10 s`

期望：

1. 页面可正常进入，无死页、无空白页、无崩溃
2. 进入该页后，不应继续表现出常规 PID 轮询主导的延迟特征
3. 红点若收到车辆数据，应开始有连续变化

失败特征：

1. 红点始终锁死在中心，且长时间无变化
2. 页面进入后明显卡顿或 BLE 断开
3. 离开该页后其他常规指标无法恢复

### 4.2 `0x0D0` 连续性

步骤：

1. 保持低速匀速直行
2. 做轻微左右转向
3. 做轻微加速与轻微松油
4. 观察 `15-30 s`

期望：

1. 红点应持续刷新，而不是只跳一次
2. 左右微转时，红点应有横向偏移
3. 轻微加减速时，红点应有纵向偏移
4. 黄区应逐步积累成不规则包络，而不是完全不变化

若不满足，优先怀疑：

1. ELM327 克隆对 `ATMA` 支持不稳定
2. 目标车当前工况下未按预期持续广播
3. 监听文本行格式和当前适配器输出格式存在差异
4. BLE notify 丢包或节流
5. 若串口里长期没有 `GFORCE OBD sample ...`，说明问题更靠近监听链路而不是 UI

### 4.3 方向符号校验

建议在安全封闭环境中做低强度动作：

1. 车辆轻微左转
2. 车辆轻微右转
3. 车辆轻微加速
4. 车辆轻微制动

期望象限：

1. 左转时，红点偏向 `LEFT` 一侧
2. 右转时，红点偏向 `RIGHT` 一侧
3. 加速时，红点偏向 `ACC` 一侧
4. 制动时，红点偏向 `BRAKE` 一侧

如果方向反了：

1. 先确认文档中象限命名和当前视觉理解一致
2. 再确认 `0x0D0` 字段字节映射是否需要符号翻转
3. 最后确认不是安装方向与预设坐标系不同造成的误解

## 5. `G-force-esp32` 验收步骤

### 5.1 静止归零

步骤：

1. 设备固定安装
2. 静止停放
3. 切到 `G-force-esp32`
4. 停留 `5-10 s`

期望：

1. 红点回到圆心附近
2. 小幅抖动被抑制，不应持续高频乱跳
3. 黄区会逐步回落

如果静止时长期偏一侧：

1. 先检查设备安装是否倾斜
2. 再检查 bias 采样时是否处于运动状态
3. 最后考虑是否需要调整 `QMI8658` 轴映射
4. 若串口里的 `raw` 正常而 `filt` 长期偏一侧，优先怀疑 bias 或轴方向配置

### 5.2 方向校验

步骤：

1. 手持或车辆低速状态下，让设备随车体产生轻微左/右侧倾或横向变化
2. 轻微加速 / 轻微制动

期望：

1. 横向变化时，红点落到正确 `LEFT / RIGHT` 象限
2. 纵向变化时，红点落到正确 `ACC / BRAKE` 象限
3. 红点移动应比 OBD 版本更灵敏，但仍有平滑感

### 5.3 重新进入页面

步骤：

1. 从 `G-force-esp32` 切到其他页
2. 再切回 `G-force-esp32`

期望：

1. IMU 链路应重新启用
2. bias 采样应重新生效
3. 页面重新进入后不应沿用明显错误的旧偏置

## 6. UI 体感验收

这一节同时适用于 `G-force-OBD` 和 `G-force-esp32`。

必须确认：

1. 圆盘吃屏充分，没有明显大片闲置区域
2. 四象限文字不会挤到圆心，也不会贴边过近
3. 红点不是“抖成噪点”，也不是“拖尾太重”
4. 黄色历史包络能体现最近一段时间的峰值分布
5. 静止后黄区会慢慢收，不会永远挂住
6. 红点柔光存在但不过度抢戏
7. 页面整体观感更接近参考图，而不是通用工程图控件

## 7. 故障分流

### 7.1 只有 `G-force-OBD` 不动

优先检查：

1. BLE 连接是否正常
2. ELM327 是否支持 `ATMA`
3. `0x0D0` 是否真的被当前车型 / 当前工况广播
4. 监听文本格式是否与 `zc6_gforce_monitor_decode.h` 的解析假设一致
5. 串口是否出现：
   - `Entered G-force OBD monitor mode for CAN 0x0D0`
   - 但始终没有 `GFORCE OBD sample ...`

### 7.2 只有 `G-force-esp32` 不动

优先检查：

1. `QMI8658` 是否初始化成功
2. 页面切换后 `qmi8658_gforce_set_enabled(true)` 是否触发
3. 板载 IMU 安装方向和代码映射是否一致
4. 串口里是否有：
   - `QMI8658 ready`
   - `QMI8658 bias ...`
   - `GFORCE IMU raw=...`

### 7.3 两个页面都能动，但方向错

优先判断：

1. 是只有 IMU 方向错
2. 还是只有 OBD 方向错
3. 还是 UI 象限命名理解与实现坐标定义不一致

不要直接同时改两条数据链，先隔离是哪一条方向错。

## 8. 通过口径

若要把当前阶段判断为“G-force 两个表真实可用”，至少应满足：

1. `G-force-OBD`：
   - 在目标车与目标 ELM327 上能持续收到有效变化
   - 离开页面后常规 PID 页面恢复正常
2. `G-force-esp32`：
   - 静止时能回到圆心附近
   - 轻微动态下方向基本正确
3. UI：
   - 红点阻尼体感可接受
   - 黄区衰减可接受
   - 四象限语义与实际体感一致

在这三组都成立前，当前状态仍应视为：

1. 代码与自测闭环已完成
2. 真机路测验收待补
