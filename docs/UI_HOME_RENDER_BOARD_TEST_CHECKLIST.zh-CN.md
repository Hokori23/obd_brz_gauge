# 首页渲染重构板端验证清单

更新时间：`2026-07-03`

## 1. 目的

用于验证本轮首页渲染重构已经达到“可上板端到端测试”的交付标准。

覆盖对象：

1. `G-force-OBD`
2. `G-force-esp32`
3. 普通 1~6 槽仪表页
4. `GEAR`

## 2. 刷机前提

1. 固件已经由当前工作区重新构建：
   - `build/obd_brz_gauge.bin`
2. ESP-IDF 环境已激活。
3. 设备串口在线。

建议命令：

```powershell
idf.py -p <PORT> flash monitor
```

## 3. G-force 页面验证

### 3.1 G-force-OBD

1. 切到 `G-force-OBD`
2. 确认页面进入后没有明显卡顿加重
3. 车辆静止时：
   - 红点应基本回到圆心
   - 黄色历史包络应逐步回落
4. 缓慢左右转向或轻微姿态变化时：
   - 红点应连续移动
   - 不应出现明显对象跳位感
5. 切出页面再切回：
   - 页面应正常恢复
   - 不应出现黑屏或残影异常

### 3.2 G-force-esp32

1. 切到 `G-force-esp32`
2. 轻微旋转表体或板子
3. 确认：
   - 红点方向与实际预期一致
   - 历史包络会跟随新的方向展开
   - 回正后历史包络会衰减

## 4. 普通仪表页验证

### 4.1 公共项

对 1~6 槽模板分别至少抽一页验证：

1. 页面分割线仍然存在
2. 切页时无新增卡顿或明显延迟
3. 数值刷新过程中：
   - 不应出现字体反复抖动
   - 不应出现 title / unit 频繁跳位

### 4.2 宽度与裁剪

优先用较长值的指标验证：

1. `RPM`
2. `SPD`
3. `BKT`
4. `BOOST`

确认：

1. 数值不会因为每次刷新重复重排而闪动
2. 重进页面后布局保持一致

## 5. GEAR 页面验证

1. 切到 `GEAR`
2. 确认：
   - 档位数字正常显示
   - 转速环正常绘制
   - 超过红线时数字与红段闪烁同步
3. 切出切回：
   - 不应出现黑屏
   - 不应出现环消失或残留

## 6. 切页与刷新联动验证

1. 在 `G-force`、普通仪表页、`GEAR` 之间来回切换
2. 确认：
   - 页面切换方向正常
   - 当前页刷新正常
   - 旧页状态不会污染新页

## 7. 判定标准

满足下面条件即可视为本轮达到板端验收入口：

1. 所有页面都能正常进入和退出
2. 没有新增崩溃、黑屏、卡死
3. `G-force` 自绘链路正常工作
4. 普通仪表页静态背景线正常显示
5. 普通仪表页刷新时无明显重复排版抖动
